# ClipForge AI

**Telugu-first AI video clipping** — turn long-form Telugu YouTube uploads into
short, captioned vertical clips ready for Reels / Shorts / TikTok.

The pipeline picks the strongest 30–60s moments from a video, crops them to
9:16 with subject tracking, and burns in word-by-word karaoke captions with
correctly-shaped Telugu conjuncts (వత్తులు / ఒత్తులు) — the shaping bug that
the naive `subtitles=` FFmpeg filter has on Telugu below-base is the reason
the caption burn is locked to `ass=` + `fontsdir`, see `BURNIN_NOTES.md`.

---

## Architecture — the 6-stage pipeline

Each job flows through six stages (`api/worker.py :: process_video`). Progress
is persisted in Redis and streamed to the frontend via polling
(`GET /jobs/{id}`) with a `current_stage` field for the UI.

| # | Stage        | What it does                                                   | Module |
|---|--------------|---------------------------------------------------------------|--------|
| 1 | **Download** | Fetch the source video (YouTube URL via yt-dlp, or uploaded file) | `services/video_downloader.py` |
| 2 | **Transcribe** | Word-level Telugu ASR — sentence & word timestamps            | `services/transcriber.py` (Sarvam ASR) |
| 3 | **Select**   | LLM picks the top viral moments with hook / payoff / virality  | `services/clip_selector.py` (Gemini) |
| 4 | **Cut**      | FFmpeg trims each selected span (multi-segment stitch supported) | `services/video_cutter.py` |
| 5 | **Crop**     | Subject-tracked 9:16 crop with smoothed centering             | `services/vertical_cropper.py` (MediaPipe) |
| 6 | **Caption**  | Word-by-word karaoke ASS + deterministic burn-in              | `services/caption_renderer.py` |

**Contracts worth knowing before touching this code:**
- **`ass=` filter only** for caption burn. `subtitles=` mis-shapes Telugu
  below-base conjuncts (`ప్ర` decomposes) on the production libass — locked
  down by `tests/test_caption_shaping.py`. See `BURNIN_NOTES.md §1`.
- **Bundled caption fonts.** `services/fonts.py :: CAPTION_FONTS_DIR` points
  libass at OFL-licensed `.ttf` files (Noto Sans Telugu / Ramabhadra / Mandali)
  so font resolution never depends on host fontconfig. The **same font files**
  drive the React caption editor preview (WYSIWYG).
- **Normalized 0–1 coords everywhere.** Frontend and API payloads stay
  normalized end-to-end; the *only* pixel conversion is
  `services/canvas_coords.py :: to_pixel_center`, so every aspect ratio (9:16 /
  1:1 / 16:9) lands `\pos` at literal output pixels.

## Tech stack

**Frontend** — React 19 + Zustand + React-Router v7 + Tailwind + shadcn/ui,
built with CRACO on top of Create React App. Zustand owns the entire editor
state (`frontend/src/store/useAppStore.js`); coords stay 0–1 through the
edit document (see `frontend/src/lib/editDocumentValidation.js`).

**Backend** — FastAPI (`api/main.py`) + Celery worker (`api/worker.py`)
+ Redis (jobs / task queue) + FFmpeg (compositing + burn-in). Jobs are kept
for 24h (Redis TTL).

**AI / ML** — Sarvam ASR (Telugu word timestamps), Gemini
(`services/clip_selector.py` — clip selection with hook/payoff scoring),
MediaPipe (`services/vertical_cropper.py` — face-tracked 9:16 crop),
sentence-transformers (semantic similarity for clip selection).

**Captions** — Word-level karaoke ASS generated by
`services/caption_renderer.py :: generate_ass_karaoke` and burned via FFmpeg's
`ass=` filter with `fontsdir=services/assets/fonts` so the exact bundled
`.ttf` files render on any machine.

## Setup

### 1. Environment variables

Copy `.env.example` (or create `.env`) at repo root with:

```
# LLM + ASR
GEMINI_API_KEY=...          # required for clip selection (services/clip_selector.py)
SARVAM_API_KEY=...          # required for Telugu ASR (services/transcriber.py)

# Optional — email notifications on job completion
RESEND_API_KEY=...
NOTIFY_FROM=noreply@example.com

# Infra
REDIS_URL=redis://localhost:6379/0
```

`frontend/.env` sets the API base for the React app:

```
REACT_APP_API_BASE=http://localhost:8000
```

### 2. Infra (Redis)

```bash
docker-compose up -d      # starts the Redis container defined in docker-compose.yml
```

### 3. Backend

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# API
uvicorn api.main:app --reload --port 8000

# Celery worker (separate terminal)
celery -A api.worker.celery_app worker --loglevel=info
```

FFmpeg is required on `PATH` for the cut / crop / burn stages.

### 4. Frontend

```bash
cd frontend
yarn install
yarn start                # http://localhost:3000
```

## WYSIWYG caption editor

The React editor is a strict what-you-see-is-what-you-get preview of the
exported burn:

- **Same font files.** The bundled OFL `.ttf` (Noto Sans Telugu default,
  Ramabhadra / Mandali) is loaded via `@font-face` on the editor page and
  handed to libass via `fontsdir` on export — byte-identical file, byte-identical
  metrics. No more “preview looked like Noto, export looked like Nirmala”
  drift across machines that don’t have Noto installed system-wide.
- **Drag with clamped bounds.** The user drags the caption on the canvas
  and the frontend clamps the resulting `(x,y)` to the measured text bounding
  box against the current video dimensions, so the caption can’t slip off
  the frame in the editor (and therefore never off the exported frame).
- **Explicit `\an5\pos(cx,cy)` for both default and dragged captions.**
  Preview and export share a single positioning code path — the export ASS
  always carries an explicit center anchor + pixel position, matching the
  preview’s `translate(-50%,-50%)` centering exactly, whether or not the user
  ever touched the caption.

Contract in code:

- `frontend/src/data/captionStylePreview.js` — the CSS/JS mirror of the 11
  backend style presets (`services/caption_renderer.py :: STYLES`). Colors,
  outlines and box treatment decoded from the raw `&HAABBGGRR` values so the
  decoder itself is testable.
- `frontend/src/components/editor/ElementBodies.jsx :: CaptionBody` — renders
  a caption line at the same `wordsPerLine` grouping as the backend
  (`buildCaptionLines` mirrors `group_words_into_lines`).
- `frontend/src/components/editor/CanvasArea.jsx` + `ElementRenderer.jsx` —
  drag / snap-to-guides / clamp-to-frame handling.

## Tests

```bash
# Backend
pytest tests/ -q

# Frontend
cd frontend && CI=true yarn test --watchAll=false
```

All caption-path changes must keep `tests/test_caption_shaping.py` and
`tests/test_caption_positioning.py` green — those guard the two invariants
that ship the actual product: correct Telugu shaping and byte-identical
preview↔export placement.
